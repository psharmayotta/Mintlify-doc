Our Inference Platform allows you to deploy and serve AI models in multiple ways depending on your infrastructure, scalability requirements, and workload patterns.

You can quickly get started with pre-built models or deploy your own custom models with full control over compute resources and scaling.

## Deployment Options

### Shared Endpoint

Shared endpoints allow you to start using AI models instantly without managing infrastructure.

**Features**

- Access multiple pre-deployed models
- Pay-as-you-go usage
- No infrastructure management
- Quick API access

**Best For**

- Prototyping
- Testing models
- Low traffic applications
- Rapid development

**Example:**

```bash
curl https://api.yourplatform.com/v1/chat/completions \
-H "Authorization: Bearer API_KEY" \
-d '{
 "model": "qwen3-14b",
 "messages":[{"role":"user","content":"Hello"}]
}'
```

### Dedicated Deployment

Dedicated deployments allow you to deploy models on isolated infrastructure with full control over configuration.

**Features**

- Dedicated GPU resources
- Custom scaling configuration
- High throughput workloads
- Production deployments

**Best For**

- Enterprise applications
- High-traffic systems
- Low latency requirements
- Private AI workloads

### Bring Your Own Cloud (BYOC)

You can deploy models directly on your own cloud infrastructure while still using our control plane.

Supported environments include:

- AWS
- GCP
- Azure
- Private Kubernetes clusters
- On-premise GPU clusters

**Benefits**

- Full infrastructure control
- Private networking
- Enterprise security compliance
- Custom scaling policies

## How Deployment Works

Deploying a model involves a few simple steps:

### 1. Select Model

Choose a model from the marketplace or upload your own model.

### 2. Configure Resources

Select the compute resources required:

- GPU type (A10G, A100, H100)
- Number of GPUs
- CPU and memory allocation

### 3. Configure Scaling

Define scaling metrics such as:

- GPU utilization
- CPU utilization
- Request queue length

These metrics automatically scale the deployment based on workload.

### 4. Deploy Model

Click **Deploy** to start the deployment process. The model will be available through API endpoints once the deployment is completed.

> \[!NOTE\] Model deployments typically take 30-60 seconds to become available.

## Monitoring and Observability

Our platform provides built-in monitoring tools for production deployments.

You can track:

- GPU utilization
- Request throughput
- Latency metrics
- Error rates
- Resource consumption

These metrics help optimize model performance and infrastructure usage.
