This guide walks you through deploying your first AI model on Q0. We'll use the multimodal Qwen 2.5 VL 7B model as an example, which can process both text and images.

## Prerequisites

*   A Q0 account ([Sign up here](https://ui-alpha.q0.dev/signup) if you haven't already)
*   Basic familiarity with the Q0 platform
*   An API key ([Generate one here](https://ui-alpha.q0.dev/api-key) if needed)

## Deployment Process

<Steps>
  <Step title="Select a Model">
    1. Navigate to the **Deployment** section in the left sidebar
    2. Search for "Llama 3.1 8B" in the search bar
    3. select the Model for the Deployment
    
    <Frame>
      <img src="/images/model-selection.png" alt="Model Selection" />
    </Frame>
  </Step>

  <Step title="Model Configuration">
    Configure the model settings according to your requirements. Choose the deployment type (e.g., Dedicated) and specify the model ID.
  </Step>

  <Step title="Resource Allocation">
    Select the appropriate GPU and compute resources for your workload.

    <Frame>
      <img src="/images/performace.png" alt="Resource Allocation" />
    </Frame>
  </Step>

  <Step title="Configure Scaling">
    Set up autoscaling rules to handle varying traffic loads efficiently.

    <Frame>
      <img src="/images/scalling.png" alt="Configure Scaling" />
    </Frame>
  </Step>

  <Step title="Deploy">
    Review your configuration and click **Deploy**. Your model will be ready in a few minutes.
  </Step>

  <Step title="Clean Up Resources">
    To avoid unnecessary charges, clean up your deployment when you’re done testing:

    1. Navigate to the **Deployments** section in the left sidebar
    2. Find and select your deployment
    3. Click the three-dots menu in the top-right corner
    4. Select **Delete**
    5. Enter the deployment name to confirm deletion
    6. Click **Delete** to complete the process

    <Note>
      This step is crucial to free up GPU resources and prevent unexpected charges.
    </Note>
  </Step>
</Steps>

## Understanding Your Deployment

Your deployed model provides an OpenAI-compatible endpoint, meaning you can use it with any client library that supports the OpenAI API format. The endpoint supports:

*   Text and image inputs (for multimodal models like Qwen VL)
*   Streaming and non-streaming responses
