## **Training models**

The Training Module wizard has 7 steps and for the selected Model and Training Configuration user can perform the trainings.

**Basic 7 Steps followed are -**

1. **Basic Details** - Provide a unique name, type, and description to identify your fine-tuning job
2. **Model Selection** – Select based on Model Categories
   1. Diffusion Model
   2. Large Language Model
   3. Vision Language Model
   4. Automatic Speech Recognition

 

3.  **Training Configuration** -Fields differ based on the selected **Training Type**: **SFT** or **RLHF**

Adjust advanced options to control how your model learns, optimizes, and performs. Fields here differ based on **Training Type**.

- The fine-tuning strategy for the LLM. Switching between `SFT` and `RLHF` changes which fields appear further down

**SFT (Supervised Fine-Tuning)** — Trains the model on labeled instruction/response pairs so it learns to reproduce that style and content directly. This is the default starting point for teaching a model new knowledge, formats, or behaviors.

**RLHF (Reinforcement Learning from Human Feedback)** — preference-based tuning on top of a fine-tuned checkpoint; adds a **DPO** parameter group.

4. **Dataset Details** – Attach and configure the dataset that will be used for training your model.

Select an existing Dataset or Create a New Dataset

5. **Evaluation** - Define evaluation settings to measure performance and validate your fine-tuned model.

**Skip validation dataset** — No portion of data is held back for evaluation; the model trains on 100% of the available data with no in-training validation metrics computed. Use this when you're confident in your hyperparameters and want to maximize the data used for training (e.g. a small dataset where every example matters), or for quick smoke-test runs where evaluation isn't needed.

**Auto-split training data** — Automatically holds out a portion of the training dataset (not used for weight updates) to compute validation loss/metrics during training. This gives you an unbiased read on whether the model is generalizing or overfitting, without requiring you to prepare a separate file. Recommended for most runs since it needs no extra setup.

**Use custom dataset** — Lets you point to a separate, purpose-built validation dataset instead of splitting the training set. Useful when you want evaluation on a curated, held-out benchmark that's distinct from (and not diluted by) your training distribution — e.g. testing OCR accuracy on a specific document type the training set doesn't fully represent.

| Field / Option | Description |
| :-- | :-- |
| Skip validation dataset | No data held back for evaluation; trains on 100% of available data. Best for quick runs or very small datasets. |
| Auto-split training data _(Recommended)_ | Automatically holds out a portion of the training data for validation, no extra setup needed. |
| Use custom dataset | Uses a separate, dedicated dataset for validation instead of splitting the training set. |

6. **Infrastructure Configuration** - Configure the compute resources and environment used for training your model

| Field | Description |
| :-- | :-- |
| Infrastructure Type | The infrastructure account training runs on, e.g. `Yotta Infrastructure Account`. |
| GPU type | `H100` (8 CPUs, 80GB HBM3, ₹432.00/GPU/hr) or `L40S` (4 CPUs, 48GB GDDR6 ECC, ₹190.80/GPU/hr). |
| Node count | `1`, `2`, `4`, or `8`. |
| GPU Count Per Node | `1`–`8`. |

7. **Optional Settings** - Modify extra settings to tailor the model's training and output to your needs

Deployment Configurations set up by selecting **Quantisation**

Selecting **Create Training** submits the job.

## After Submission — Training Status

Identical structure for every model category: a details page with **Details**, **Usage**, and **Logs** tabs.

| Tab | What it shows |
| :-- | :-- |
| Details | Training Status stepper (`Training Received` → `Downloading Data` → `Training Started` → `Training Completed` → `Saving Weights` → `Completed`), General Details panel, and expandable sections for Dataset Details, Evaluation, Infrastructure Configuration, and Train Configuration. **Deploy** becomes available once training completes. |
| Usage | Request ID and Job ID. |
| Logs | Streaming training logs, filterable by level, time range, and timezone. |
