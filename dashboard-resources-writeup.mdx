## Why this section exists

Most people don't come to an inference platform wanting an API key — they come wanting to _ship something_. The **Resources** panel on the Q0 Dashboard exists to close that gap: four opinionated, production-ready starting points that take you from "I have a model" to "I have a working app" in the time it takes to read this page.

Think of it less as a links widget and more as a **launchpad** — each card is a fast-path into one of the four things people building on Q0 most often want to build: an autonomous agent, a conversational chatbot, a knowledge-grounded RAG app, or a real-time image generation experience.

## Build an Agent

**What it is:** A starter workflow for building an AI system that doesn't just answer — it _acts_. The line between a chatbot and an agent comes down to autonomy: a system that only answers questions is a chatbot, but one that's connected to tools and takes action on a user's behalf is an agent, and it can range from a single tool call to a full multi-agent system orchestrating several specialized agents together.

**Why it matters:** 2024–2026 has been the shift from "LLM as autocomplete" to "LLM as operator." Agents are the difference between a model that tells you how to file an expense report and one that actually files it.

**What you can build with it:**

- A research assistant that searches the web, pulls sources, and drafts a summary — unattended
- An internal ops agent that reads a ticket, checks a knowledge base, and takes the next action (create a task, notify a team, escalate)
- A customer-facing agent that can actually resolve a request (refund, reschedule, update a record) instead of just describing how to do it
- A coding agent that plans a task, writes code, and iterates on test failures

**How Q0 helps:** Once you've fine-tuned or picked a base model on Q0, this resource shows you how to wire that model into an agent loop — perceive, decide, act, repeat — using the deployed endpoint as the reasoning engine.

## Build a Next.js Chatbot

**What it is:** A production-ready conversational interface, scaffolded in Next.js, that streams responses from your Q0-hosted model straight into a polished chat UI.

**Why it matters:** The chatbot is still the most common front door for AI products — but the gap between "a chat demo" and "a chat _product_" is entirely in the plumbing: streaming tokens, message history, error states, loading UX, auth. This resource handles that plumbing so you can focus on what the bot actually says.

**What you can build with it:**

- A support widget embedded on your marketing site or app
- An internal tool for your team — a "Slack-but-smarter" assistant scoped to company workflows
- A branded assistant for a specific product surface (onboarding helper, docs assistant, in-app copilot)
- A rapid prototype to demo a fine-tuned model to stakeholders before committing to a full build

**How Q0 helps:** Point the template at any of your Q0 model deployments — a base model, or one you've fine-tuned through the Training Module — and you have a shippable chat surface without hand-rolling the streaming and state layer yourself.

## Build RAG Application

**What it is:** A retrieval-augmented generation starter — the pattern of grounding a model's answers in your own documents instead of its training data alone. In practice, RAG means embedding your source material, retrieving the most relevant chunks for a given question via semantic similarity, and passing those chunks to the model alongside the query so it can answer using content it was never trained on.

**Why it matters:** Every model has a knowledge cutoff and no visibility into your private data. RAG is how you hand a general-purpose model your company's actual knowledge — policies, product docs, support history — without the cost of retraining it. It's the difference between a model that says "I don't have information on that" and one that answers correctly, citing your own source material.

**What you can build with it:**

- A support bot that answers from your actual product docs, policies, and past tickets — not generic web knowledge
- An internal "second brain" search assistant over your team's wiki, meeting notes, and specs
- A compliance or legal Q&A tool grounded strictly in approved source documents
- A customer-facing product assistant that stays current as your docs update, without any model retraining

**How Q0 helps:** This resource pairs a Q0-hosted embedding \+ generation model with a vector retrieval step, so you can go from "a folder of PDFs" to "a working, grounded Q&A app" in one guided build.

## Build a Real Time Image App

**What it is:** A starter app for real-time image generation, powered by FLUX — turning a text prompt (or an existing image) into a new image in the time it takes to hit enter, fast enough to feel interactive rather than batch-processed.

**Why it matters:** Image generation used to mean submitting a job and waiting. The current generation of distilled diffusion models can produce sub-second, production-grade generations, which is what unlocks genuinely _interactive_ image products — sliders, live previews, and prompt-as-you-type experiences — rather than "submit and wait" batch tools.

**What you can build with it:**

- A live product-visualization tool (e.g. "see this couch in five colors" or "preview this room in a new style")
- A creative playground where users iterate on a prompt and watch results update as they type
- An on-brand asset generator for marketing or social teams — no design software required
- A personalization feature: user-uploaded photos transformed into a specific style or theme in real time

**How Q0 helps:** This resource is wired to a fast diffusion model deployment on Q0 (Stable Diffusion or FLUX-class models), with the low-latency serving path already handled — so "real time" is a property of the app you get, not something you have to build yourself.
